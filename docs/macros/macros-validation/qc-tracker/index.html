%qc_tracker

Created by Jonathan Amato

Descriptionâ€‹

qc_tracker does the following:

  • refreshes the qc_tracker by setting all the comparison datasets together from %proc_compare
  • creates qc_tracker_subset dataset with the applied filters from optional macro parameters

Defintionâ€‹

%macro qc_tracker(
    name = null,
    mismatch = null,
    topline = null,
    before = null,
    on = null,
    after = null,
    debug = 0
);

Required Parametersâ€‹

None

Optional Parametersâ€‹

nameâ€‹

Add subset by your name.

mismatchâ€‹

Add subset to outputs that aren't matching

toplineâ€‹

Add subset by topline outputs.

beforeâ€‹

Add subset to outputs last run before the specified ISO Date

onâ€‹

Add subset to outputs last run on the specified ISO Date

afterâ€‹

Add subset to outputs last run after the specified ISO Date

debugâ€‹

Set to 1 if you want qc_tracker to preserve all datasets created in the macro for debugging purposes.

Examplesâ€‹

get all my programs that are mismatching for topline that were ran on 2024-08-10

%qc_tracker(
  name     = FLastName,
  mismatch = Y,
  on = 2024-08-10,
  topline  = Y
);

Outcomeâ€‹

ProjectFolder/Output/QC/Tracker/qc_trackerâ€‹

Refreshes qc_tracker SAS dataset and excel file that keeps track of all the relevant QC information across all programs

qc_tracker_subsetâ€‹

Creates qc_tracker_subset dataset with the applied filters from optional macro parameters
